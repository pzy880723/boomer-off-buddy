
# BOOMER OFF 品牌管理后台 — 首期 UI 框架方案

首期目标是搭出**全部模块的可点击 UI 框架**，使用模拟数据展示效果，让你能完整走通"采购 → 入库 → 成本核算 → 调拨 → 销售 → 加盟商"的页面流。后端启用 Lovable Cloud（数据库 + Server Functions 已就绪），但本期不接入真实有赞/爬虫/物流 API。

## 视觉与布局

经典 ERP 风格：浅色主题 + 固定左侧可折叠侧边栏 + 顶部面包屑/全局搜索 + 主内容区为数据表/卡片。

```text
┌──────────────────────────────────────────────────────┐
│ [Logo] BOOMER OFF                  搜索  通知  头像  │
├──────────┬───────────────────────────────────────────┤
│ 仪表盘    │  面包屑：首页 / 采购管理 / 日本大宗        │
│ 采购物流▾ │  ┌─────────────────────────────────────┐  │
│  日本大宗 │  │  KPI 卡片 / 数据表 / 图表             │  │
│  日本小包 │  │                                       │  │
│  国内渠道 │  │                                       │  │
│ 商品库存▾ │  └─────────────────────────────────────┘  │
│ 门店加盟▾ │                                           │
│ 知识库    │                                           │
│ 系统设置  │                                           │
```

设计 token：浅灰底（`oklch(0.985 0 0)`）+ 深灰文字 + 品牌主色（暖橙 `oklch(0.68 0.16 50)`，呼应"中古杂货铺"复古质感），辅以涨/跌色（绿/红）、标签色。全部写在 `src/styles.css`，组件使用语义 token。

## 各页面内容

**仪表盘 `/dashboard`**
- 顶部 4 个 KPI 卡：今日营收、本月营收、在库总估值、全渠道订单数（含同比箭头）
- 采购渠道占比（环形图）+ 物流成本趋势（折线图）
- 批次回本率监控（进度条列表，>100% 高亮盈利色）
- 门店销售排行（横向柱状图 Top 10）
- 图表使用 recharts（已在 shadcn chart 中）

**采购管理**
- 日本大宗：表格列出大宗包裹（包裹号、关联订单数、日元金额、汇率、运费、关税、状态）；详情抽屉显示订单明细 + 成本分摊预览
- 日本小包裹：订单列表（EMS 单号、商品、金额、轨迹状态徽章）
- 国内渠道：顶部 Tabs（闲鱼/抖音/小红书/拼多多），下方为"已抓取订单"表格 + "手动同步"按钮（mock）
- 物流追踪：包裹列表 + 点击查看时间线轨迹

**商品与库存**
- 商品档案：表格（SKU、RFID EPC 占位、名称、分类、批次、估算成本、零售价、状态）+ 搜索/筛选 + 新增/批量导入按钮
- 采购批次：批次卡片网格，每张卡显示总成本、预期营收、当前营收、**回本率进度条**、商品数；点击进入批次详情（商品列表 + 成本分摊编辑）
- 库存调拨：调拨单列表 + 新建调拨弹窗（源仓 → 目标门店 + 商品多选）

**门店加盟**
- 门店列表：表格（类型徽章 直营/加盟、名称、加盟商、有赞门店 ID、地址、状态）
- 加盟商管理：卡片网格（头像、姓名、合同期、关联门店数、物料/装修进度条）
- 有赞对接：连接状态卡片 + "同步商品/拉取订单"按钮（mock 触发）+ 同步日志

**知识库**
- 左侧分类树（商品知识/SOP/QA/装修流程）+ 右侧文章列表 + 详情查看（富文本预览占位）

**系统设置**
- 三个 Tab：角色权限、数据字典、操作日志（均为占位表格 + "即将推出"提示）

## 数据层

启用 Lovable Cloud。本期**不建表**，避免和后续真实业务模型冲突——所有页面用 `src/lib/mock-data.ts` 里的内联模拟数据驱动（采购订单、批次、商品、门店等）。当你确认业务模型时，再按 PRD 第 4 章建表并替换为真实 Server Functions 调用。

## 技术细节

- 侧边栏使用 shadcn `Sidebar` 组件，`collapsible="icon"`，跟随当前路径高亮
- 表格使用 shadcn `Table` + 顶部筛选栏；分页/排序为前端 mock
- 图表使用 shadcn `chart`（recharts 封装）
- 所有交互按钮（新增/导入/同步）连接到 `toast` 提示"功能开发中"，避免空白点击
- 所有路由文件设置独立 `head()` 元数据（title/description）
- 中文 UI 文案

## 不包含（后续迭代）

- 登录/权限（你确认暂不需要）
- 真实有赞 API、平台爬虫、物流 API 对接
- RFID 硬件接入
- 真实数据库表与 CRUD（仅启用 Cloud 待用）
- AI 智能估价

## 实现步骤

1. 启用 Lovable Cloud
2. 配置设计 token（`src/styles.css`）+ 全局布局（`__root.tsx` 侧边栏 + 头部）
3. 创建 mock 数据文件
4. 按模块顺序创建路由：仪表盘 → 采购 → 库存 → 门店 → 知识库 → 设置
5. 移除首页占位，`/` 重定向到 `/dashboard`
